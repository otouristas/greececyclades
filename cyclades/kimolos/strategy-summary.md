# Kimolos Island: Two-URL Content Strategy
## Executive Summary & Implementation Guide

---

## The Strategy at a Glance

You asked: *"How would I split content between `/islands/kimolos` and `/guides/kimolos` and how would they connect?"*

**Answer:** Create a **Hub & Spoke model** where:

- **`/islands/kimolos`** = Quick reference gateway (4,000-5,000 words)
- **`/guides/kimolos`** = Comprehensive planning resource (12,000-15,000 words)
- Both pages link strategically to **conversion pages** (`/ferries`, `/hotels`, `/flights`)

---

## Why This Strategy Works

### For SEO:
✓ **2x Content Footprint:** Two pages compete for different keywords
- Hub ranks for: "Kimolos Island" (broad, high-volume)
- Guide ranks for: "Best beaches in Kimolos" (specific, long-tail)

✓ **No Duplicate Content:** Each page is 40-60% unique from the other
- Hub summarizes; Guide deep-dives
- Different examples, different angles, different context

✓ **Strategic Authority Distribution:** Links build authority on hub, then funnel to conversions

### For Users:
✓ **Choice of Depth:** Quick readers get hub; planners get guide
✓ **Clear Navigation:** Hub tells you "want details?" with link to guide
✓ **Multiple Entry Points:** Different keywords lead to different pages
✓ **Easy Conversions:** Both pages link to booking/conversion pages

### For Conversions:
✓ **Awareness → Interest → Consideration → Action**
✓ Hub gets awareness traffic → guides to detailed planning → books ferries/hotels

✓ **Two Conversion Paths:**
- Path A: Hub → Guide → Ferries/Hotels
- Path B: Direct to Guide → Ferries/Hotels

✓ **Authority Building:** More content + strategic linking = more referral authority to conversion pages

## Common Questions Answered

**Q: Won't two pages cannibalize each other in search?**
A: No. They target different keywords (broad vs. long-tail) and different user intents. Google understands the hub summarizes and the guide provides details.

**Q: How much content overlap is OK?**
A: 20-40% overlap is fine (same topic, different depth). Make sure anchor text and examples differ.

**Q: What if someone searches for "Kimolos restaurants"?**
A: They'll likely land on `/guides/kimolos#cuisine` (guide). The hub page doesn't try to rank for specific topics.

**Q: Should the pages link to each other?**
A: Yes, strategically. Hub → Guide (8-10 links). Guide → Hub (1-2 links only). Don't create loops.

**Q: What about mobile visitors?**
A: Ensure links are clickable on mobile. Use descriptive anchor text that makes sense on all devices. Test thoroughly.

**Q: How often should I update these pages?**
A: Hub = quarterly updates; Guide = as needed (seasonal updates, new restaurants, etc.)
