## 4. Canonical URLs

### Canonical URL Generation Logic

The component includes advanced canonical URL handling to prevent duplicate content issues.

**Key Features:**
- Removes all `/index.html` references
- Enforces trailing slash consistency (root only)
- Removes trailing slashes from all other URLs
- Handles absolute vs relative URLs
- Defaults to homepage if not provided

**Implementation:**
```typescript
const getCanonicalUrl = () => {
  if (canonical) {
    let url = canonical.startsWith('http') 
      ? canonical 
      : `https://hotelssantorini.gr${canonical.startsWith('/') ? canonical : `/${canonical}`}`;
    
    // CRITICAL: Remove ALL index references
    url = url.replace(/\/index\.html?$/i, '');
    url = url.replace(/\/index$/i, '');
    url = url.replace(/index\.html?$/i, '');
    url = url.replace(/index$/i, '');
    
    // Ensure trailing slash consistency
    if (url !== 'https://hotelssantorini.gr' && url !== 'https://hotelssantorini.gr/') {
      url = url.replace(/\/$/, '');
    }
    
    // Ensure root has trailing slash
    if (url === 'https://hotelssantorini.gr') {
      url = 'https://hotelssantorini.gr/';
    }
    
    return url;
  }
  return "https://hotelssantorini.gr/";
};
```

### Canonical URL Best Practices

1. **Always include canonical tag** - one per page
2. **Use absolute URLs** - always include protocol and domain
3. **Remove query parameters** - unless they change content significantly
4. **Consistent trailing slashes** - choose one pattern and stick to it
5. **Self-referencing** - canonical should point to itself (or canonical version)

4. **Image sitemap** - separate sitemap for images
5. **Submit to Search Console** - Google Search Console, Bing Webmaster Tools

## 10. Hreflang Tags

### Hreflang Implementation

Hreflang tags support international SEO for multi-language sites.

**Supported Languages:**
- English (en)
- Greek (el)
- French (fr)
- Italian (it)
- German (de)
- Swedish (sv)
- Russian (ru)
- Turkish (tr)
- x-default (default language)

**Implementation:**
```html
<link rel="alternate" hrefLang="en" href="https://hotelssantorini.gr/page" />
<link rel="alternate" hrefLang="el" href="https://hotelssantorini.gr/el/page" />
<link rel="alternate" hrefLang="x-default" href="https://hotelssantorini.gr/page" />
```

### Hreflang Best Practices

1. **Self-referencing** - include hreflang tag for current language
2. **x-default** - always include for fallback
3. **Consistent URLs** - use consistent pattern across languages
4. **Canonical compatibility** - hreflang and canonical can coexist
5. **Valid language codes** - use ISO 639-1 codes
6. **Same page, different languages** - all language versions link to each other

## 11. Image SEO

### Image Optimization Strategy

**File Naming:**
- Descriptive, keyword-rich filenames
- Hyphens, not underscores
- Lowercase letters
- Example: `oia-sunset-view-hotel.jpg`

**Alt Text:**
- Descriptive and contextual
- Include relevant keywords naturally
- Describe what's in the image
- Avoid keyword stuffing
- Example: "Luxury cave hotel with infinity pool overlooking Santorini caldera in Oia"

**Image Sitemap:**
- Separate XML sitemap for images
- Include all important images
- Proper image metadata (title, caption, geo location)

**Technical Requirements:**
- **Format:** WebP (modern) or JPG/PNG (fallback)
- **Dimensions:** Optimized for use case (responsive)
- **File size:** Compressed (use tools like TinyPNG)
- **Lazy loading:** Implement for below-fold images
- **Responsive images:** Use srcset for different screen sizes

### Image Schema Markup

For important images, include ImageObject schema:

```json
{
  "@context": "https://schema.org",
  "@type": "ImageObject",
  "url": "https://hotelssantorini.gr/uploads/hotel-image.jpg",
  "caption": "Luxury cave hotel with infinity pool",
  "contentLocation": {
    "@type": "Place",
    "name": "Oia, Santorini"
  }
}
```

## 13. Keyword Strategy

### Keyword Research Methodology

**Primary Keywords:**
- High-volume, high-intent keywords
- Main category terms (e.g., "santorini hotels")
- Location + keyword combinations

**Secondary Keywords (LSI):**
- Related semantic keywords
- Long-tail variations
- Question-based keywords
- Local intent keywords

**Keyword Density:**
- **Primary keyword:** 1-2% of content
- **Secondary keywords:** Natural distribution
- **Avoid stuffing** - maintain readability

### Keyword Placement

1. **Title tag** - Primary keyword in first 60 chars
2. **H1** - Include primary keyword naturally
3. **First paragraph** - Keyword in first 100 words
4. **H2/H3 tags** - Include LSI keywords
5. **Meta description** - Primary + 1-2 secondary
6. **Alt text** - Relevant keywords when appropriate
7. **URL slug** - Include primary keyword
8. **Internal links** - Use keyword-rich anchor text

### Keyword Generation Function

The SEO component includes automatic keyword generation based on page type:

```typescript
const generateOptimizedKeywords = (): string[] => {
  const baseKeywords = ['santorini hotels 2026', 'greece accommodation', 'cyclades islands'];
  
  const pageSpecificKeywords = {
    'homepage': ['best santorini hotels 2026', 'luxury santorini resorts', ...],
    'hotels': ['compare santorini hotels', 'santorini hotel deals 2026', ...],
    'location': locationData ? [`${locationData.name.toLowerCase()} hotels`, ...] : [],
    // ... more page types
  };
  
  return [...baseKeywords, ...(pageSpecificKeywords[pageType] || [])];
};
```

## 14. Performance & Technical SEO

### Core Web Vitals

**Largest Contentful Paint (LCP):**
- Optimize images (WebP, lazy loading)
- Minimize render-blocking resources
- Use CDN for assets
- Optimize server response times

**First Input Delay (FID):**
- Minimize JavaScript execution time
- Code splitting and lazy loading
- Defer non-critical JavaScript
- Use Web Workers for heavy tasks

**Cumulative Layout Shift (CLS):**
- Set image dimensions (width/height)
- Reserve space for dynamic content
- Avoid inserting content above existing content
- Use CSS aspect-ratio for responsive images
